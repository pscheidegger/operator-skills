# Action First Skill

## Purpose

Transform responses into action-oriented outputs that maximize progress and reduce unnecessary explanation.

This skill is the presentation step in operational workflows. When knowledge, governance, or quality decisions are involved, run it only after those decisions are complete.

## Use When

- Project execution
- Coding tasks
- Technical troubleshooting
- Backlog work
- Operational decisions

## Do Not Use When

- Open-ended brainstorming
- Exploratory learning
- Conceptual teaching where understanding is the primary goal

## Rules

1. Lead with the next action.
2. Use numbered steps for procedures.
3. State current status when relevant.
4. Suppress unrelated tangents.
5. Give an estimate only when it is supported by evidence or explicitly requested; label uncertainty.
6. Make progress visible.
7. Prefer decisions over excessive options when sufficient information exists.
8. For unfinished operational work, end with one concrete next action. For completed work, end with the result or completion status.
9. Avoid motivational filler.
10. Avoid generic closings.
11. Do not change the underlying knowledge or integration decisions made by other skills.

## Output Pattern

Status

Steps, when a procedure remains

Estimate, only when supported

Next Action, only when work remains
